# Factory vs Abstract Factory: вибір

Патерни Factory Method і Abstract Factory належать до породжувальних патернів. Вони потрібні для того, щоб правильно створювати об’єкти в програмі й не прив’язувати код до конкретних класів. Це робить програму більш гнучкою та зручною для розширення.

# Різниця між Factory Method і Abstract Factory

Factory Method — це патерн, який дозволяє створювати один тип об’єкта, але конкретний клас цього об’єкта визначається у підкласі. Тобто є метод для створення об’єкта, але саме підклас вирішує, що саме створювати.

Abstract Factory — це патерн, який використовується для створення цілої групи пов’язаних об’єктів. Тобто не один об’єкт, а одразу набір об’єктів, які підходять один до одного.

# Коли використовувати Factory Method

Цей патерн підходить, коли:

1.ми не знаємо наперед, який саме об’єкт потрібно створити;

2.потрібно, щоб підкласи самі визначали тип створюваного об’єкта;

3.система може мати різні варіанти одного продукту.

Наприклад, є програма для створення різних типів документів: PDF і Word. Базовий клас має метод створення документа, а конкретні підкласи створюють потрібний тип документа.

# Коли використовувати Abstract Factory

Abstract Factory краще використовувати тоді, коли:

1.потрібно створювати групу пов’язаних об’єктів;

2.важливо, щоб ці об’єкти були сумісні між собою;

3.система має кілька варіантів “наборів” об’єктів.

Наприклад, створення інтерфейсу для Windows і macOS. Для кожної системи потрібні свої кнопки, меню та інші елементи. Abstract Factory дозволяє створити повний набір елементів для конкретної платформи.

# Переваги та недоліки

Factory Method простіший у реалізації та легко дозволяє додавати нові типи об’єктів. Але може з’явитися багато підкласів.

Abstract Factory зручний для великих систем, бо дозволяє змінювати цілі набори об’єктів. Але він складніший у реалізації, і якщо потрібно додати новий тип продукту, доведеться змінювати багато коду.

# Висновок
Отже, Factory Method краще підходить для створення одного об’єкта з різними варіантами реалізації. Abstract Factory використовується, коли потрібно створювати одразу групу пов’язаних об’єктів. Вибір залежить від складності програми та вимог до її розширення.
