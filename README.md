## Контрольні питання
## Питання 1: Які типи винятків найчастіше виникають при роботі з файлами та мережею?
При роботі з файлами найчастіше виникають FileNotFoundException, DirectoryNotFoundException та IOException. Для мережевих операцій типовими є HttpRequestException, SocketException та TimeoutException. Ці винятки з’являються при нестабільному інтернеті, недоступності сервера або заблокованих/відсутніх файлах.

## Питання 2: Поясніть принцип роботи патерну Retry. Коли його доцільно використовувати?
Патерн Retry автоматично повторює виконання операції, якщо сталася тимчасова помилка. Програма виконує операцію, ловить помилку, перевіряє, чи можна її повторювати, робить паузу і пробує знову. Retry доцільно використовувати для тимчасових проблем, наприклад, нестабільне з’єднання, тимчасово зайнятий файл або недоступний сервер. Його не слід застосовувати, якщо помилка постійна або повтор виконання небезпечний.

## Питання 3: Як реалізувати експоненційну затримку між повторними спробами?
Експоненційна затримка між спробами означає, що час очікування збільшується в геометричній прогресії. Формула: delay = initialDelay * 2^(attempt-1). Це дозволяє не перевантажувати ресурс і давати йому час на відновлення.

## Питання 4: Для чого потрібен делегат shouldRetry у допоміжному класі RetryHelper?
Делегат shouldRetry у RetryHelper визначає, чи слід повторювати операцію для конкретного винятку. Він дозволяє контролювати повтори, роблячи Retry тільки для помилок, які є тимчасовими, і пропускаючи критичні або постійні помилки.

## Висновок

У лабораторній роботі було реалізовано обробку помилок вводу/виводу та мережевих помилок за допомогою try-catch, а також універсальний RetryHelper з експоненційною затримкою та делегатом shouldRetry. Це дозволяє повторювати операції лише для тимчасових помилок, підвищує надійність програми та робить застосунок більш відмовостійким. Використання патерну Retry демонструє, як ефективно працювати з нестабільними ресурсами та забезпечувати стабільність виконання операцій.
